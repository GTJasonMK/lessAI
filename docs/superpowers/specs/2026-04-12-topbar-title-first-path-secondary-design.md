# 顶部栏标题优先与路径次级化设计

## 背景

当前顶部栏中，标题、路径、状态 chip 放在同一视觉层级内。即使已经改成真正的省略号显示，长路径仍会和标题争夺注意力，使整体观感显得拥挤、生硬。

## 目标

- 顶部栏以文档标题为主视觉。
- 文件路径下沉为标题下方的次级说明。
- 路径在未超出时完整显示，超出时仅使用尾部省略号。
- 不调整右侧按钮组的位置与尺寸。
- 不引入字符串级裁剪或中间省略逻辑。

## 范围

仅调整工作区顶部栏中部信息区：

- 文档标题
- 文件路径显示层级与样式

不调整：

- 右侧操作按钮与窗口控制按钮
- 状态、模型、应用信息的文案含义
- 其他面板、列表、卡片中的标题展示方式

## 最终方案

采用 `C2` 方案：标题优先，路径下沉为次级信息。

### 布局

- 中部信息区改为纵向两层：
  - 第一层：文档标题
  - 第二层：路径说明
- 状态 chip、模型、应用等信息继续保留在相邻区域，不与路径混排在同一行内。

### 标题

- 保持现有字号和主视觉身份。
- 使用完整原文。
- 仅通过 CSS `text-overflow: ellipsis` 在超出时显示省略号。
- 悬停 `title` 显示完整标题。

### 路径

- 从强调性的胶囊信息中降级为次级说明文本。
- 字号更小、颜色更淡，弱于标题。
- 使用完整清理后路径。
- 仅通过 CSS `text-overflow: ellipsis` 做尾部省略。
- 未超出时完整显示。
- 悬停 `title` 显示完整路径。

## 结构建议

- `WorkspaceBar` 中保留 `rawTitle` 和 `rawPath`。
- 标题单独使用可省略文本层。
- 路径使用独立的次级文本层，不再作为强调性 `context-chip` 的主体内容。
- 路径前缀 `路径：` 保留，但应与路径文本共同作为一行次级说明处理。

## 样式原则

- 路径不能再与标题同粗细、同边框权重。
- 省略号必须由文本层负责，而不是由整个容器负责。
- 不新增会影响按钮排布的宽度策略。
- 不通过手工截断字符串制造“伪省略”。

## 测试

- 回归断言需覆盖：
  - 标题文本层存在且使用 `ellipsis`
  - 路径次级文本层存在且使用 `ellipsis`
  - 不存在字符串级 `formatTopbarTitle` / `formatTopbarPath`
- 运行：
  - `node scripts/ui-regression.test.mjs`
  - `pnpm run typecheck`

## 风险与控制

- 风险：中部信息区从单行变成双层后，垂直节奏会变化。
- 控制：仅在中部信息区内部做层级调整，不改右侧按钮区与全栏主栅格结构。
