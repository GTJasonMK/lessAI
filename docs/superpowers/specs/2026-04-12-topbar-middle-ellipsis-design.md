# 顶部栏标题与路径中间省略设计

## 背景

当前顶部栏中的文档标题和文件路径使用的是尾部省略。对于长标题和长路径，这会造成“文本被硬截断”的观感，用户难以快速判断末尾关键信息，尤其是文件名后半段和路径末级目录。

## 目标

- 将顶部栏中的文档标题改为中间省略。
- 将顶部栏中的文件路径改为中间省略。
- 保持顶部栏现有布局、按钮尺寸、chip 排列不变。
- 悬停时仍可查看完整标题和完整路径。

## 范围

仅调整工作区顶部栏中的以下显示：

- 文档标题
- `路径：...` 文本

不调整以下内容：

- 顶部栏整体布局
- 操作按钮宽度、位置和图标
- 其他 chip 的文本策略
- 侧边栏、文档面板、会话列表中的标题显示

## 方案

采用字符串级中间省略，而不是继续依赖 CSS 尾部省略。

### 标题

- 保留标题前后两端的信息。
- 中间以单个省略号字符连接。
- 当标题足够短时，原样显示。

示例：

- 原文：`04-3 作品报告（大数据应用赛，2025版）模板.docx`
- 显示：`04-3 作品报告（大数…2025版）模板.docx`

### 路径

- 先沿用现有的显示路径清理逻辑，例如移除 Windows `\\?\` 前缀。
- 在清理后的路径上做中间省略。
- 优先保留盘符或根路径感，以及末尾文件名。
- 中间目录折叠为省略号。

示例：

- 原文：`E:\Code\LessAI\testdoc\04-3 作品报告（大数据应用赛，2025版）模板.docx`
- 显示：`E:\Code\LessAI\testdoc\…\2025版）模板.docx`

## 实现约束

- 不引入基于像素测量的动态裁剪。
- 不新增 ResizeObserver 或文本宽度测量逻辑。
- 仅在顶部栏组件中使用新的中间省略格式化结果。
- 保留 `title` 提示为完整原文。

## 测试

- 为标题中间省略补前端工具函数测试或最小静态断言。
- 为路径中间省略补前端工具函数测试或最小静态断言。
- 运行 `pnpm run typecheck`。
- 运行 `node scripts/ui-regression.test.mjs`，确认没有引入顶部栏回归。

## 风险与控制

- 风险：按字符数裁剪时，视觉宽度不是绝对最优。
- 控制：本次只解决“硬截断感”和“末尾信息不可见”问题，保持实现简单且稳定。
