# LessAI 综合测试（Markdown）

> 建议分别切到 `段落 / 整句 / 小句`，然后观察 Markdown 语法是否保留可见、正文是否可读、保护区是否只锁定结构本身

## 1. 文档说明与测试目标

这一份样本用于覆盖标题、列表、链接、图片、行内样式、代码块、数学块、表格、HTML 注释、脚注和引用标记。理想行为是：普通正文可处理，语法骨架保持原样，改写后仍能拼回合法 Markdown。

### 1.1 快速检查点

- [x] 标题语法仍可见。
- [x] 行内代码、数学、链接和图片语法被保护。
- [x] 代码块、表格、Front Matter、引用定义等块级结构不会被错误改写。

## 6. 行内强调与常见样式

这里集中放一些行内结构：**重点结论**、*补充说明*、`cargo fmt --check`、$E=mc^2$、[@ref-demo]、[^note1]，以及 `<span data-role="mark">行内 HTML 标签</span>`。

理想行为是：语法标记本身保持锁定，语法包裹的自然语言部分仍然能按当前适配器的规则参与阅读或拆分。
